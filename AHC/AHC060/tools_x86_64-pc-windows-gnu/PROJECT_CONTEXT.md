# AHC060 氷クリーム配達最適化 - プロジェクト概要

## 問題概要

- グラフ上でN個のノード（K個の店 + N-K個の木）
- 店は最初位置0にある
- 目標：異なるアイスクリームのパターンを各店に配達して、ユニークな配達数を最大化
- 時間制限：T=2000ターン
- アイス：赤(R)と白(W)の2色のみ

## main.cpp（問答無用配達）の詳細実装

### パラメータ構造体

```cpp
struct Params {
    int rProb;      // R変換確率：1/rProb の確率で変換
    int minLen;     // 最小コーン長さ
    int invCap;     // 在庫キャパシティ基準値
    int invBoost;   // コーン長さに応じた在庫許容の係数
    int treeBias;   // コーンが短い時に木を優先する確率(%)
};
```

**最適値：{50, 1, 14, 2, 75}**

### コア配達ロジック（simulate関数内）

```
1. ランダムに隣接ノードを選択
2. コーン長 < minLenなら、treeBiasの確率で木を優先
3. 【重要】店を通った時、持っているアイスを店が持っていなければ即座に配達
   - 在庫が少ない店から優先
   - 条件判定（長さチェック）なし
4. R変換：1/rProbの確率で実行
```

### 重要な改善点

- **条件判定の削除**：以前は`coneLen >= lenLong`などで配達を制限していた
  - これが**無駄にコーンを長くして、ターンを消費**していた
  - 削除後、スコアが大幅改善(旧659.8 → 新692.8点)
- **treeBias=75**：短いコーン時に木を優先して、スムーズにアイスを集める

## 別AIからの技術的発見・分析（2026-02-01追記）

### 乱数シードの固定について（重要）

main.cppの`simulate()`内で `mt19937 rng(12345)` と乱数シードが固定されている。

- **影響**: 同じ入力・同じパラメータでは常に同じ結果になる
- **SAの最適化対象**: 「固定シードでの最適パラメータ」を探索している
- **懸念点**: 本番の複数テストケースで頑健でない可能性
- **改善案**:
  - SAのループ内で複数シードを試して平均スコアで評価
  - または入力ファイルのハッシュを乱数シードに使用

### 現在のアルゴリズムの弱点分析

1. **局所的な判断のみ**: 隣接ノードからの選択で、数ステップ先を見ていない
2. **パターンの重複リスク**: 短いコーン（W, R, WW等）は店間で被りやすい
3. **グラフ構造の未活用**: 店・木の配置やクラスタ構造を考慮していない

### 具体的な改善アイデア

1. **2-3手先読み**: 現在位置から2-3ステップ先までシミュレートして最善手を選ぶ
2. **店への最短経路事前計算**: BFSで各ノードから最寄りの店への距離を持っておく
3. **コーンの希少性判定**: 「このコーンを配達できる店は何店舗あるか」を見て、希少なら急いで配達
4. **木のR/W分布の活用**: Rが少ない領域では戦略的にR変換を実行

## 【実験報告】main_v3.cpp / main_v4.cpp - 目的地指向型移動（別AI）

### 目的

ランダムウォークを改善し、「目的地に向かって移動する」戦略を試した。

### main_v3.cpp（失敗）

**戦略**: BFSで最短距離を事前計算し、常に目的地方向へ移動

- コーンあり → 配達可能な最寄りの店へ
- コーンなし → 最寄りの木へ

**結果**: 平均634点（ベースライン682点より悪化）

**失敗理由**:

- 貪欲すぎて途中の機会を逃す
- 問答無用配達と干渉

### main_v4.cpp（改善版・成功）

**戦略**: 「緩やかな誘導」- 即配達できない時のみ、配達可能な店方向へ緩やかに誘導

```cpp
struct Params {
    int rProb;      // R変換確率
    int minLen;     // 最小配達長さ
    int treeBias;   // 木優先確率
    int guideBias;  // 店方向への誘導確率 ← 新パラメータ
};
```

**ロジック**:

1. 問答無用配達が最優先（隣接に配達可能な店があれば即配達）
2. 配達先がない場合のみ、guideBias%の確率で配達可能な店方向へ誘導
3. それ以外はランダム

**結果（10シード平均）**:
| バージョン | 平均スコア |
|-----------|-----------|
| main.cpp | 682 |
| main_v4.cpp | **692** (+10) |

**最適パラメータの傾向**:

- minLen = 5-6（元の1より大きい）
- guideBias = 10-30%（控えめな誘導が最適）
- treeBias = 55-100%（木を優先）

### 重要な教訓

1. **貪欲すぎはNG**: 常に目的地へ直行するより、ランダム性を残す方が良い
2. **問答無用配達との相性**: 即配達を優先し、それ以外の時だけ誘導
3. **minLenの再考**: 誘導システムありの場合、minLen=5-6が最適（長めのコーンを作る方が良い）
